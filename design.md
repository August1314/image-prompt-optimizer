# Design Direction

## Goal
Create a premium promotional landing page for Image Prompt Optimizer inspired by the editorial rhythm, confident typography, and calm product storytelling of bauhausclock.com, while keeping the visual identity original and tailored to this product.

## Positioning
Image Prompt Optimizer is a local-first prompt workflow tool for AI image generation. The page should communicate:

- fast prompt refinement
- clarity without complexity
- local privacy and optional desktop power features
- support for OpenAI and Gemini
- mock mode for local development
- macOS desktop availability

## Creative Principles

1. Premium but restrained
   - Use large type, strong spacing, and simple layouts.
   - Avoid clutter and over-decoration.

2. Editorial pacing
   - Build the page in stacked sections with varied scale.
   - Alternate between statement blocks, feature blocks, and proof blocks.

3. Product-first clarity
   - Every section should answer one question: what it does, why it matters, how it works, why trust it.

4. Calm confidence
   - Use soft gradients, subtle borders, and a dark neutral base.
   - Accent color should feel modern and technical, not playful.

5. Originality
   - Reference the overall feel of a high-craft landing page, but do not copy layout, copy, or visual motifs directly.

## Visual Direction

### Layout
- Full-width hero with a centered content column
- Modular sections with large spacing between them
- Gentle vertical rhythm, like a long-form product story
- Sticky or floating call-to-action in desktop view if useful

### Color System
- Background: near-black or charcoal
- Surface: slightly lighter panels
- Text: warm white with muted secondary copy
- Accent: electric violet or cyan-violet blend
- Success and warning states kept subtle

### Typography
- Use a strong sans-serif stack
- Very large hero headline
- Short supporting paragraphs
- Section labels in uppercase or small caps style
- Keep line lengths comfortable

### Motion
- Minimal motion only
- Fade and slide-in on section reveal is acceptable
- CTA hover states should feel polished
- Avoid excessive parallax or distracting animation

## Page Structure

1. Hero
   - Clear product claim
   - One-sentence value proposition
   - Primary and secondary CTA
   - Small credibility line

2. Social proof / trust band
   - Key facts: local-first, desktop runtime, provider support
   - Optional testimonial-style statements

3. Feature narrative
   - 3–5 premium feature cards
   - Focus on prompt clarity, provider flexibility, privacy, and desktop workflow

4. Workflow section
   - Show the flow: idea → clarification → optimized prompt → export/use
   - Make it feel fast and deterministic

5. Desktop / reliability section
   - Explain keychain storage, local settings, license flow, and mock fallback behavior

6. FAQ / objections
   - Answers to privacy, supported providers, desktop requirements, and mock mode

7. Final CTA
   - Reinforce the value proposition
   - End with a direct action

## Copy Tone
- Concise
- Confident
- Technical where helpful
- No hype language
- No vague claims

## Assets
- Prefer pure CSS, gradients, and typography over heavy image assets
- If imagery is used later, it should be sparse and integrated as part of the story

## Acceptance Criteria
- The page feels like a polished marketing site rather than an app shell
- The user can understand the product in under 30 seconds
- The page is responsive on desktop and mobile
- The design remains original while capturing the refined editorial mood
- Content matches the actual product capabilities
